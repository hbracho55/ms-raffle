Creación de nuevo SP para visualizar transferencias

Detalles del problema y propuesta de solución para la optimización del proceso.

Situación actual

  • El procedimiento actual encargado de buscar transferencias pendientes utiliza un SP que obtiene información de transferencias en una tabla que presenta inconsistencias.
  • No existe algún SP que permita buscar transferencias con filtro de estados de TEF: Rechazada (REC), Eliminada (ELI), Procesada (PRP).
  • Se utilizan varios SP para obtener información completa de las transferencias, lo cual es ineficiente.

Solución Propuesta

La propuesta de solución consiste en la creación de un nuevo SP que permita obtener información consistente y que además disponga distintos filtros que permita mayor flexibilidad al momento de solicitar información


Algunos filtros necesarios:

  • Estado.
  • Cuenta origen.
  • Cuenta destino.
  • Rut de apoderado.
  • Rut de destinatario.

Fases propuestas para optimización del proceso de TEF:

  1. Fase 1: Implementación en nuevas funcionalidades: Utilizar el nuevo SP en funcionalidades para la descarga de TEF a Excel y para el Panel de Transferencias, permitiendo visualizar TEF con estados Eliminada, Firmadas y Finalizadas.
  2. Fase 2: Corrección y optimización: Utilizar el nuevo SP para obtener transferencias pendientes, corrigiendo la inconsistencia de datos que posee el SP anterior y optimizando los tiempos de respuesta.
  3. Fase 3: Estandarización de la información: Implementar el nuevo SP en todas las funcionalidades que requieran obtener información de transferencias para mantener la misma estructura de datos y evitar la duplicidad de lógicas.
  4. Fase 4: Optimización con Redis: Se cargará la información de transferencias en Redis para reducir las consultas directas a la base de datos, mejorando significativamente el performance de los flujos.
